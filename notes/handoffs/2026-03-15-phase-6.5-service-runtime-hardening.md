# Handoff: Phase 6.5 — Service Runtime Hardening

**Date**: 2026-03-15
**Role**: Implementer

## What Was Done

Prevents service SIGTERM by capping per-run signal processing and
extending the systemd timeout.

Key changes:
- `runAggregation` takes a new `Int` parameter for max signals; applies
  `take maxSignals` to ready signals after gate checks pass. Deferred
  signals remain on disk for the next timer tick. Logs when capping.
- `--max-signals N` CLI flag (default 20) added to `ccs aggregate`.
- `TimeoutStartSec = "30min"` added to systemd service (was default
  90s, insufficient for LLM-heavy processing).
- `maxSignals` option added to Nix module, wired to `--max-signals`
  in `aggregateArgs`.

Files changed:
- `src/CCS/Aggregate.hs`: `runAggregation` signature + `take` logic
- `app/Main.hs`: `acMaxSignals` field, `--max-signals` flag, wiring
- `nix/modules/home-manager/project-status-sync/default.nix`:
  `TimeoutStartSec`, `maxSignals` option, CLI flag wiring
- `test/Main.hs`: updated 2 `runAggregation` call sites
