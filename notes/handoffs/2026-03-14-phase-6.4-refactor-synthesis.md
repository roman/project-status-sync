# Phase 6.4 continuation: refactor generateStatusForProject

**Date**: 2026-03-14
**Session**: 635a9395

## What changed

Refactored `generateStatusForProject` from a deeply nested procedural function
into a pipeline of pure gate functions with bag types, plus migrated test JSON
from escaped strings to `aeson-qq`.

### New types

- `SynthesisPaths` — filesystem paths for synthesis artifacts (was inline `let`)
- `Watermark` — opaque newtype over Int for cursor position (was raw Int)
- `SynthesisSkip` — why synthesis was skipped (`NoEvents` | `UpToDate`)
- `SynthesisWork` — gate output: incremental flag + new events
- `SynthesisHistory` — handoff files + previous STATUS.md from disk
- `SynthesisContext` — resolved prompt-ready context

### New pure functions

- `decideSynthesis` — pure gate: `[EventLogEntry] -> Watermark -> Either SynthesisSkip SynthesisWork`
- `resolveContext` — pure: combine work + history into context (2 branches, not 3)
- `buildSynthesisInput` — pure: context -> prompt text

### Other changes

- `gatherHistory` uses Applicative composition for independent IO reads
- `readTextFile`/`listHandoffs` inlined as `where` bindings in `gatherHistory`
- Test JSON in `parseEventsJsonl` tests uses `aesonQQ` quasi-quotes + `strictJsonl` helper
- 13 new unit tests for the pure functions (128 total)

## Review process

Two review agents were run in parallel:
- **socratic-naming (Opus)**: drove renames `ProjectArtifacts→SynthesisPaths`,
  `Cursor→Watermark`, flagged `SynthesisHistory` as borderline
- **code-critic**: identified `resolveContext` branch collapse, over-extracted
