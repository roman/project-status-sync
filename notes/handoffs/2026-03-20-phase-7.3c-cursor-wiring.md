**Date**: 2026-03-20
**Status**: COMPLETE

## What Was Done

Wired the extraction cursor into `processSession` so that re-processed
sessions only extract events from new transcript lines, preventing
duplicate events in EVENTS.jsonl.

## Implementation

### Cursor file format (evolved from 7.3a)

7.3a stored a single int (format "TBD by 7.3c"). Now stores a JSON map
keyed by session ID: `{"session-abc": 42, "session-def": 100}`.

- `readExtractionCursors :: FilePath -> m (Map SessionId ExtractionCursor)`
- `writeExtractionCursors :: FilePath -> Map SessionId ExtractionCursor -> m ()`
- Negative values filtered at parse boundary
- Missing/corrupt file → empty map (safe fallback)

### processSession control flow

1. Compute `projectDir` and `cursorFile` path early (moved up from inside `withLLMResult`)
2. Read cursor map, look up session → `cursorPos` (0 on miss)
3. Read transcript as lazy ByteString, pass to `filterTranscriptFrom cursorPos`
4. If filtered is empty AND `newLineCount > cursorPos` → save cursor (advances past non-extractable lines)
5. If filtered is non-empty → run extraction → on LLM success: save cursor, write events, handoff, progress
6. On LLM failure → cursor NOT saved → lines re-processed next run

### Code critic fix

The critic identified that the empty-filtered path didn't save the cursor,
causing infinite re-processing of non-extractable lines. Fixed by adding
`saveCursor` on the empty-filtered path when `newLineCount > cursorPos`.
