## Key Decisions

- **Script packaging**: Standalone `writeShellApplication` derivation (not inline
  in the Nix module). Follows the beads pattern for consistency and testability.
- **Module location**: Lives in this repo (`nix/modules/home-manager/ccs-session-end-hook/`),
  exported as `homeManagerModules.ccs-session-end-hook` via nixDir. Zoo.nix integration
  is out of scope for this task.
- **Option namespace**: `programs.claude-code.plugins.conversation-sync` (follows the
  existing `programs.claude-code.plugins.*` convention from haskell-development-skill).
- **Hook mechanism**: Uses `programs.claude-code.settings.hooks.SessionEnd` (the
  settings.json hooks object), not the `programs.claude-code.hooks` option (which
  writes raw script files to `~/.claude/hooks/`).
- **Script source**: Move logic from `scripts/session-end-hook.sh` into the package
  derivation directly.
