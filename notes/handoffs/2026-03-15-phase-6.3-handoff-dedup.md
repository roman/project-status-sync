# Handoff: Phase 6.3 — Handoff Dedup via Prior Filenames

**Date**: 2026-03-15
**Role**: Implementer

## What Was Done

`generateHandoff` now reads the handoff directory listing before invoking the
LLM and prepends filenames as "Previous handoffs in this project:" to the
prompt input metadata. The handoff-generation prompt instructs the LLM to
suppress redundant content when prior handoffs already cover the same topic.

Added empty-content guard: if the LLM returns only a TOPIC line (no markdown
body), the handoff file is not written, preventing empty files from
accumulating in the handoff directory.

Files changed:
- `src/CCS/Process.hs`: `generateHandoff` reads handoff dir, prepends prior
  filenames to metadata, skips writing empty content. Removed duplicate
  `handoffDir` binding (reuses one defined earlier in scope).
- `prompts/handoff-generation.md`: Added optional prior filenames input,
  added dedup quality criterion.
