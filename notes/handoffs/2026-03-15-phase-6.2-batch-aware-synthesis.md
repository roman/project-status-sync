# Handoff: Phase 6.2 — Batch-Aware Synthesis

**Date**: 2026-03-15
**Role**: Implementer

## What Was Done

Moved status synthesis out of the per-session loop so it runs once per
project at the end of an aggregation batch, eliminating the N×
redundant LLM calls observed in the production audit.

Key changes:
- `processSession` returns `Maybe Project` instead of `()`, and no
  longer calls `generateStatus`. Returns `Nothing` for non-git
  sessions, `Just project` when processed (even on empty transcript
  or extraction failure, so synthesis still picks up prior events).
- `AggregateResult` is now generic (`AggregateResult a`) with
  `AggregatedSessions ![a]`, keeping `CCS.Aggregate` decoupled from
  `CCS.Project`. `runAggregation` collects callback results via
  `forM` instead of `forM_`.
- New `generateStatusForProject` in `CCS.Process` takes `Project`
  directly (no `AvailabilitySignal`), derives paths from project key.
- `app/Main.hs` extracts touched projects via `catMaybes`, deduplicates
  by `ProjectKey` with `nubBy`, then calls `generateStatusForProject`
  once per unique project.

Files changed:
- `src/CCS/Aggregate.hs`: generic `AggregateResult a`, `forM` in
  `processAll`, `Show a` constraint on `runAggregation`
- `src/CCS/Process.hs`: `processSession` return type, removed old
  `generateStatus`, added `generateStatusForProject`
- `app/Main.hs`: dedup logic, per-project synthesis after aggregation
- `test/Main.hs`: adapted `runAggregation` test to generic type
