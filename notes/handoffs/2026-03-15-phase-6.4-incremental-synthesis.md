# Handoff: Phase 6.4 — Incremental Synthesis with Watermark Cursor

**Date**: 2026-03-15
**Role**: Implementer

## What Was Done

Implemented incremental STATUS.md synthesis using a watermark cursor so
only new events are fed to the LLM prompt on subsequent runs.

Key changes:
- `formatEventsCompact` groups events by (date, session) with markdown
  headers and bullet format, replacing raw JSONL input (~60-70% smaller).
- `parseEventsJsonl` parses EVENTS.jsonl using ByteString line splitting
  (no Text round-trip that could corrupt invalid UTF-8).
- `.last-synthesized` cursor file stores event count as integer.
  `readCursor` returns `Maybe Int` — missing/corrupt/out-of-range resets
  to `Nothing`, caller logs and falls back to full synthesis.
- `generateStatusForProject` reads cursor, slices events, reads previous
  STATUS.md for incremental mode. Falls back to full mode (all events)
  if STATUS.md is missing during incremental synthesis.
- `--full-resync` CLI flag forces cursor to 0 for manual recovery.
- Synthesis prompt updated with "Incremental vs Full" section.

Code critic fixes applied:
- **Bug**: incremental mode with deleted STATUS.md would feed only new
  events to a "generate from scratch" prompt, losing historical context.
  Fixed: falls back to full event set when STATUS.md is missing.
- **Correctness**: `parseEventsJsonl` worked on ByteString directly via
  `BS.split 0x0A` instead of Text decode/re-encode round-trip.
- **Observability**: `readCursor` returns `Maybe Int`, caller logs
  "Cursor missing or invalid" on reset for operational visibility.

Files changed:
- `src/CCS/Process.hs`: all new functions + rewritten `generateStatusForProject`
- `app/Main.hs`: `acFullResync` field, `--full-resync` flag
- `prompts/status-synthesis.md`: incremental vs full section
- `test/Main.hs`: tests for `formatEventsCompact`, `parseEventsJsonl`
