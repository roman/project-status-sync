# Phase 7.3b: Transcript Slicing in CCS.Filter

**Date**: 2026-03-20
**Status**: COMPLETE
**Role**: Implementer

## What Was Done

Added `filterTranscriptFrom` to `CCS.Filter` — the core function for
incremental transcript extraction. It drops already-processed JSONL lines
(by count) and filters/formats only the new ones.

## Spec Compliance

WORKPLAN item: "7.3b: Transcript slicing (`src/CCS/Filter.hs`)"

- Drops already-processed lines: met (`drop skipLines`)
- Returns updated cursor for storage: met (returns total line count)
- `filterTranscriptFile` becomes convenience wrapper: met (calls `filterTranscript` which calls `filterTranscriptFrom 0`)

## Implementation

```haskell
filterTranscriptFrom :: Int -> LBS.ByteString -> (Text, Int)
```

Uses `Int` (not `ExtractionCursor`) because `ExtractionCursor` lives in
`CCS.Process` which imports `CCS.Filter`. The caller in `processSession`
wraps/unwraps the newtype.

`filterTranscript` is now `fst . filterTranscriptFrom 0` — a pure refactor
preserving identical behavior.

## Key Invariant

The cursor tracks "total lines in transcript file." This is correct only
because session JSONL files are append-only (Claude Code only appends via
`--continue`). If files were ever truncated, the cursor would be invalid
and fall back to full extraction (handled by 7.3c).
