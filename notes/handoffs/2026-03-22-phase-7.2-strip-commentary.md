# Phase 7.2: Prevent LLM Commentary in STATUS.md

**Date**: 2026-03-22
**Status**: COMPLETE
**Role**: Implementer

## What Was Done

Two fixes to prevent LLM preamble/commentary from leaking into STATUS.md output:

1. **Prompt fix (7.2a)**: Added "Output Rules" section to `prompts/status-synthesis.md`
   instructing the LLM to output only the markdown document, no preamble, commentary,
   or code fences.

2. **Extraction fix (7.2b)**: Rewrote `stripCodeFences` in `src/CCS/Process.hs` to find
   the first opening fence and last closing fence, then extract only the content between
   them. Previous implementation only stripped fences when they were the first/last lines;
   any surrounding text was preserved verbatim.
